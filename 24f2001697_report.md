## 1. Introduction

### What is the problem?
This competition involves multi-class classification of chest X-ray images into 20 diagnostic categories. Given a chest X-ray image, the model must predict exactly one condition from the 20 possible classes including diseases such as Atelectasis, Cardiomegaly, Effusion, Pneumonia, and a "No Finding" class indicating a healthy scan.

### Why is it important?
Automated chest X-ray diagnosis is a critical problem in healthcare. Radiologists are often overwhelmed with large volumes of scans, and an accurate automated system can assist in prioritizing urgent cases, reducing diagnostic errors, and improving patient outcomes especially in resource-limited settings.

### What is the dataset?
The dataset consists of chest X-ray images in PNG format with the following structure:
- **Training set:** 51,043 images with 20 binary label columns (one-hot encoded)
- **Test set:** 17,015 images (labels hidden)
- **Image format:** PNG, grayscale converted to RGB
- **Label type:** Single-label multi-class (exactly one condition per image)

The 20 classes are: Atelectasis, Cardiomegaly, Consolidation, Edema, Effusion, Emphysema, Fibrosis, Hernia, Infiltration, Mass, Nodule, Pleural Thickening, Pneumonia, Pneumothorax, Pneumoperitoneum, Pneumomediastinum, Subcutaneous Emphysema, Tortuous Aorta, Calcification of the Aorta, and No Finding.

### What is the objective?
To train a deep learning model that maximizes the competition score defined as the mean per-class asymmetric score: **Score_c = (TP - FP - 5×FN) / N_c**, averaged over all 20 classes. The heavy penalty on False Negatives (×5) means missing a disease is much more costly than a false alarm.

## 2. Methodology

### Models Used

#### Primary Model: Swin Transformer Tiny (swin_tiny_patch4_window7_224)
The Swin Transformer was chosen as the primary model because:
- It uses **hierarchical shifted windows** which capture both local and global features — critical for detecting small lesions and nodules in X-rays
- Unlike plain ViT which treats all patches equally, Swin's hierarchical design preserves spatial relationships
- It has shown superior performance on medical imaging benchmarks compared to CNN-based models
- The Tiny variant fits within T4 GPU memory constraints (16GB)

#### Baseline Model: ResNet34
Used as a baseline for comparison:
- Lightweight and fast to train
- Well-suited for establishing a performance floor
- Used plain CrossEntropyLoss with class weights

### Preprocessing Steps

**One-hot to Single Label Conversion:**
The original labels are one-hot encoded. These were converted to single class indices using argmax. Images with no positive label were assigned to the "No Finding" class.

**Image Resizing:**
All images were resized to 224×224 pixels — the native resolution for Swin Transformer's patch size of 4×4 with window size 7.

**Normalization:**
ImageNet normalization was applied (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]) since pretrained weights were trained on ImageNet.

### Techniques Used

**Data Augmentation (Training):**
- Random horizontal flip (p=0.5)
- Random rotation (±10°)
- Color jitter (brightness=0.2, contrast=0.2)
Applied only during training to improve generalization.

**Transfer Learning:**
Both models used ImageNet pretrained weights. This gave a strong initialization and significantly reduced training time compared to training from scratch.

**Weighted Random Sampler:**
To handle class imbalance, each training sample was assigned a weight inversely proportional to its class frequency. This ensures rare classes like Hernia and Fibrosis are sampled as frequently as common ones like No Finding.

**Class-Weighted Loss:**
CrossEntropyLoss was used with per-class weights computed as: weight_c = total_samples / (N_classes × count_c). This further compensates for class imbalance in the loss calculation.

**Regularization:**
- Dropout (p=0.4) in the classification head
- Gradient clipping (max_norm=1.0) for training stability
- Weight decay (0.05) in AdamW optimizer

**Learning Rate Schedule:**
- 3-epoch linear warmup followed by cosine annealing decay
- This prevents the transformer from diverging early and ensures smooth convergence

**Test Time Augmentation (TTA):**
7 inference passes with random horizontal flips and rotations were averaged to produce more stable predictions.

**Per-Class Scale Tuning:**
After training, a grid search was performed on the validation set to find the optimal probability scale factor per class. This helped improve the competition score by boosting underconfident classes.

## 3. Experiments and Results

### Experiment 1: Baseline ResNet34
- **Config:** IMG=224, BATCH=32, LR=1e-4, Epochs=20, StepLR(step=7)
- **Result:** Best Val CompScore = -6.53 | Kaggle Public = -6.40
- **Observation:** Model overfitted after epoch 16 (train loss=0.01 but val score plateaued)

### Experiment 2: ResNet34 with Competition-Aware Loss
- **Config:** Same as above but replaced CrossEntropyLoss with custom CompetitionLoss
- **Result:** Best Val CompScore = -8.37
- **Observation:** Custom loss was unstable, converged slower than CrossEntropy

### Experiment 3: Swin Transformer Tiny
- **Config:** IMG=224, BATCH=16, LR=5e-5, Epochs=25, Warmup+CosineAnnealing
- **Result:** Best Val CompScore = -5.12 | Kaggle Public = -5.06
- **Observation:** Swin-T outperformed ResNet34 by 1.4 points on the public leaderboard

## 4. Conclusion

### Summary of Approach
The final solution used a pretrained Swin Transformer Tiny model fine-tuned on the chest X-ray dataset with:
- Class-weighted CrossEntropyLoss to handle imbalance
- Weighted random sampling for balanced mini-batches
- AdamW optimizer with warmup + cosine annealing schedule
- 7-pass Test Time Augmentation for stable inference
- Per-class probability scale tuning on the validation set

### Key Learnings
1. **Model architecture matters more than loss function** — Swin-T's hierarchical attention outperformed ResNet34 even with a simpler loss
2. **Stable training is critical** — dual learning rates and freeze/unfreeze strategies caused catastrophic forgetting; training the full model from the start with a low LR was more reliable
3. **The competition metric heavily penalizes FN×5** — weighted sampling and class-weighted loss are essential, not optional
4. **TTA provides consistent gains** — averaging 7 passes improved the public score from -5.12 to -5.06
5. **Early stopping prevents overfitting** — ResNet34 overfitted badly after epoch 16; patience-based stopping is crucial
